# Инструкция по редактированию текстов сайта Studio19.03

Этот документ объясняет, как править тексты раздела **Портфолио** и **страниц проектов** в файле `portfolio-and-projects.md`.

После правок вы отправляете обновлённый файл разработчику — он загрузит изменения на сайт.

---

## Какой файл редактировать

**Файл:** `portfolio-and-projects.md`  
(лежит в папке `content` проекта)

В нём собраны все тексты:
- заголовки и кнопки страницы «Портфолио»;
- названия и описания всех 18 проектов.

---

## Где открыть файл

Подойдёт любой из вариантов:

| Способ | Как |
|--------|-----|
| **Google Документы** | Скопируйте весь текст из файла → вставьте в новый документ на [docs.google.com](https://docs.google.com). После правок скопируйте обратно в файл или пришлите документ разработчику. |
| **Microsoft Word** | Откройте `.md` как текстовый файл или скопируйте содержимое. |
| **Блокнот / TextEdit** | Откройте файл напрямую. На Mac в TextEdit: Формат → «Сделать обычным текстом». |
| **Cursor / VS Code** | Если у вас есть доступ к проекту — откройте файл в редакторе. |

Расширение файла должно остаться **`.md`** (или отправьте разработчику текст в любом виде — он перенесёт в нужный формат).

---

## Главное правило

**Меняйте только сами тексты. Не трогайте служебные строки** — от них зависит автоматическая загрузка на сайт.

### Можно менять

Текст **после двоеточия** `:`

Пример — было:
```
title: Интерьеры, которые стали частью жизни своих владельцев
```

Стало:
```
title: Интерьеры, в которых хочется жить каждый день
```

### Нельзя менять, удалять и переставлять

1. **Заголовки разделов** — строки, начинающиеся с `##`:
   ```
   ## portfolio
   ## project: airhaus
   ## project: french64
   ```
   Часть `airhaus`, `french64` и т.д. — это внутренние имена проектов. Их менять нельзя.

2. **Названия полей** — слова до двоеточия:
   ```
   title:
   subtitle:
   about.task:
   about.solution:
   materials:
   ```
   Можно менять только текст **после** двоеточия (или в строках ниже — см. ниже).

3. **Разделители** — строки из трёх дефисов:
   ```
   ---
   ```

4. **Маркеры списка материалов** — дефис в начале строки:
   ```
   - Белый матовый МДФ
   ```
   Меняйте текст после `-`, сам дефис оставьте.

## Структура файла

### 1. Раздел «Портфолио» (в начале файла)

```
## portfolio

title: Заголовок страницы портфолио
subtitle: Подзаголовок под заголовком
more: Текст на кнопке карточки проекта («Смотреть проект»)
ctaTitle: Заголовок блока внизу страницы
cta: Текст кнопки внизу страницы («Обсудить проект»)
```

### 2. Каждый проект — отдельный блок

Между проектами стоит строка `---`.

```
## project: french64

title: FRENCH 64
type: Квартира
city: Москва
area: 64 м²
style: Современная классика
year: 2021
duration: 6 месяцев

about.task:
Текст о задаче проекта. Может быть несколько предложений.

about.solution:
Текст о решении — как подошли к проекту.

about.result:
Текст о результате — чем закончился проект.

materials:
- Краска · тёплый белый
- Паркет · ёлка
- Мрамор · столешница
- Латунь · свет
```

## Что означает каждое поле проекта

| Поле | Где видно на сайте |
|------|-------------------|
| `title` | Название проекта в портфолио и на странице проекта |
| `type` | Тип объекта (Квартира, Коммерция и т.д.) |
| `city` | Город |
| `area` | Площадь (например `64 м²`). Если площади нет — оставьте строку пустой: `area:` |
| `style` | Стиль интерьера в блоке фактов |
| `year` | Год реализации |
| `duration` | Срок работ (например `6 месяцев`) |
| `about.task` | Первый абзац «О проекте» |
| `about.solution` | Второй абзац «О проекте» |
| `about.result` | Цитата / итог в ленте фото и в боковом блоке |
| `materials` | Подписи к фото материалов (список через `-`) |

---

## Многострочный текст

Для полей `about.task`, `about.solution`, `about.result` текст пишется **на следующих строках** после пустого двоеточия:

```
about.task:
Задачей было создать спокойный интерьер для семьи с двумя детьми.
Нужно было вместить рабочие зоны и сохранить ощущение простора.
```

Не ставьте двоеточие и текст на одной строке, если абзац длинный — так проще читать и править.

---

## Пустые поля

Если площадь не указана:
```
area:
```
(после `area:` ничего не пишем)

Чтобы **очистить** любое поле — оставьте его пустым после `:` или удалите весь текст в блоке под ключом.

---

## Что в этом файле НЕ редактируется

Следующее на сайте настраивается отдельно — **не ищите этого в файле**:

- фотографии проектов;
- подписи к отдельным фото в ленте («Гостиная», «Кухня»);
- кнопки меню, «Следующий проект», «Все проекты»;
- расширенные тексты проекта SLOW 19 между фотографиями (уникальная вёрстка).

Если нужно изменить и это — напишите разработчику отдельно.

---

## Порядок проектов

Проекты в файле идут в том же порядке, что и на странице портфолио.  
**Менять порядок блоков в файле не нужно** — на сайте порядок не изменится.

## Чек-лист перед отправкой

- [ ] Не удалены строки `## portfolio` и `## project: …`
- [ ] Не переименованы поля (`title`, `about.task` и т.д.)
- [ ] Сохранены разделители `---` между проектами
- [ ] В списке `materials:` у каждой строки есть `-` в начале
- [ ] Проверена орфография и пробелы
- [ ] Файл сохранён или текст скопирован обратно в `portfolio-and-projects.md`

## Что сделать после правок

1. Сохраните файл `portfolio-and-projects.md`.
2. Отправьте его разработчику (почта, Telegram, Google Диск — как удобно).
3. Напишите, что тексты готовы к загрузке на сайт.

Разработчик применит изменения и при необходимости опубликует обновление.

---

## Пример: как выглядит одна правка

**Было:**
```
about.result:
Квартира кажется больше своей площади и собрана без компромиссов по эстетике.
```

**Стало:**
```
about.result:
Пространство визуально расширилось, а интерьер остался цельным и спокойным.
```

Строка `about.result:` не меняется — меняется только текст под ней.
